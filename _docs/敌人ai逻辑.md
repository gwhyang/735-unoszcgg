## 核心思路

敌人运动： 有一个speed， 直接方向*speed来运动
## 共同状态

### 巡航

玩家较远时，在自己的区域附近游走。
逻辑：ready时记录下出生点
以出生点为中心的方形内，随机选取点作为免目标点，如果选的点是没有导航层的则重新选
这个“半径”会随着节点存在的时间逐渐扩大

### 追击

玩家进入警戒范围后，按自身规则计算目标点。
所有路径计算利用导航功能，导航层为0
这些自身规则是有敌人类型决定的

### 避让

玩家速度超过击杀阈值时，不贴脸追击，优先拉开距离。

### 死亡

（这个在原本的逻辑里面已经处理）

## 敌人类型

### 1. 直追型

作用：持续施压。

目标点：

```text
Target = 玩家位置
```

行为：

一直朝玩家靠近。

适合作为最基础警船。

---

### 2. 拦截型

作用：堵玩家前路。

目标点：

```text
Target = 玩家位置 + 玩家速度方向 * 预判距离
```
相似的，如果target点没有导航层，增加预测距离重试，失败一定次数后直接以玩家位置为target

---

### 3. 包夹型

作用：和直追型形成夹击。

需要参考一个直追型敌人。

目标点：

```text
预判点 = 玩家位置 + 玩家速度方向 * 短预判距离
Target = 预判点 + (预判点 - 直追型敌人位置)
```

行为：

目标点会被另一只敌人的位置影响。

看起来更不稳定，适合制造意外包围。

---

### 4. 忽近忽远型

作用：打乱节奏。

规则：

```text
距离玩家较远：Target = 玩家位置
距离玩家较近：Target = 自己的巡航点
```
